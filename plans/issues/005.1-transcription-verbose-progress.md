# Issue 005.1 — Verbose Transcription Progress

## Goal

Make long-running transcription feel observable without compromising automation-friendly CLI output. When `clipper transcribe` is run with `--verbose`, show progress and lifecycle diagnostics on stderr for Whisper model loading/download and transcription progress, while keeping stdout reserved for human command results or parseable JSON envelopes.

## Depends On

- Issue 005

## Tasks

- Add a small shared progress/logging abstraction for CLI operations that writes only to stderr and is quiet unless verbose mode is enabled.
- Thread the existing CLI verbosity setting into the transcription command/library boundary without making core transcription logic depend directly on argparse.
- Emit verbose lifecycle messages for transcription, including:
  - resolved video name and source path
  - Whisper model name, device, compute type, and language mode
  - model loading start
  - a clear note that the first run may download model files from Hugging Face and may be slow
  - transcription start
  - segment count, detected language, duration, and transcript output path when complete
- Keep `--json` stdout strictly parseable JSON. All verbose logs and progress indicators must go to stderr.
- Add approximate transcription progress behind `--verbose` by consuming the faster-whisper segment iterator incrementally and updating progress based on `segment.end / duration`.
- Ensure progress display degrades cleanly when stderr is not an interactive TTY:
  - no animated/control-character progress bar in non-TTY contexts
  - optional plain verbose milestone lines are acceptable
- Use a lightweight progress implementation, preferably `tqdm`, only if it materially simplifies robust stderr progress behavior; otherwise implement a minimal stderr progress helper.
- Avoid changing default behavior for non-verbose runs: no progress bars or lifecycle chatter should appear unless `--verbose` is passed.
- Preserve existing `--reuse` behavior: reused transcripts should not show model loading or transcription progress, but may log that the existing transcript was reused when verbose.
- Preserve existing failure behavior and error envelopes. If model loading or transcription fails, verbose output may show the last lifecycle/progress step, but the command must still return the existing human or JSON error shape.
- Do not introduce real Whisper downloads into default tests.
- Add tests proving:
  - non-verbose transcription remains quiet apart from the final command result
  - verbose transcription writes lifecycle/progress information to stderr
  - `--json --verbose` writes valid JSON to stdout and logs only to stderr
  - progress is based on yielded segment end times and reaches completion for mocked faster-whisper output
  - reused transcripts do not load the model and do not show transcription progress
  - non-TTY stderr does not emit animated progress control sequences
- Document verbose transcription progress in `README.md`, including that model download/transcription progress appears on stderr and that first model use may download from Hugging Face.

## Acceptance Criteria

- `uv run clipper transcribe VIDEO --verbose` shows useful stderr lifecycle output before potentially slow Whisper model loading/download and during transcription.
- `uv run clipper transcribe VIDEO --json --verbose` keeps stdout parseable as a single JSON envelope while writing verbose logs/progress only to stderr.
- Non-verbose transcription output remains unchanged and does not show progress bars or lifecycle chatter.
- Progress reporting is tested with mocked faster-whisper output and does not require real model downloads in the default test suite.
- Reuse mode remains fast and does not instantiate/load Whisper when a valid transcript is reused.
- README explains how to use `--verbose` to observe long-running transcription and clarifies that first model use may download model files from Hugging Face.
